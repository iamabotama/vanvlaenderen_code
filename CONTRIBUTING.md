# Contributing to vanvlaenderen_code

Thank you for your interest in contributing! This document outlines the process for contributing to this repository.

## Getting Started

1. Fork the repository on GitHub.
2. Clone your fork locally:
   ```bash
   git clone https://github.com/<your-username>/vanvlaenderen_code.git
   cd vanvlaenderen_code
   ```
3. Create a new branch for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Workflow

- Keep commits small and focused on a single concern.
- Write clear, descriptive commit messages in the imperative mood (e.g., "Add deploy script for GitHub Pages").
- Reference any relevant issue numbers in your commit messages or pull request description.

## Pull Request Process

1. Ensure your branch is up to date with `main` before opening a pull request.
2. Provide a clear description of what your pull request does and why.
3. Link any related issues using GitHub's closing keywords (e.g., `Closes #42`).
4. A maintainer will review your pull request and may request changes before merging.

## Code Style

- Follow existing conventions in the codebase.
- Use consistent indentation (2 spaces for JavaScript/TypeScript, 4 spaces for Python).
- Comment non-obvious logic clearly.

## Reporting Issues

Please use the [GitHub Issues](https://github.com/iamabotama/vanvlaenderen_code/issues) tracker to report bugs or request features.

## Code of Conduct

Be respectful and constructive in all interactions. This project follows a standard open-source code of conduct.
